# FastAPIについて

FastAPIを触って、「今の時点で分かっていること」「正直まだ曖昧なところ」もありますが纏めました。</br>
理解が浅い部分も多く、間違っている可能性もある前提です。

## FastAPIとは

* Pythonで **Web APIを作るためのフレームワーク**
* 型ヒントを書くだけで色々やってくれる
* Swagger（APIドキュメント）が自動生成されるのが強い

## FastAPIを使うメリット

* **高速**：非同期処理（async / await）を前提に設計されている
* **型安全**：Pythonの型ヒントをそのままAPI仕様として利用
* **自動ドキュメント**：Swagger UI / ReDoc が自動生成される
* **学習コストが低い**：シンプルな記述でAPIが作れる

## FastAPIの大まかな基本構造

* `FastAPI()` : アプリケーション本体
* デコレータ（`@app.get`）でルーティングを定義
* 戻り値は自動的に JSON に変換される

## 自動ドキュメント

FastAPIは起動するだけで以下の画面が利用できる。

* `/docs` : Swagger UI
* `/redoc` : ReDoc

Swaggerから

* パラメータ入力
* リクエスト送信
* レスポンス確認

までできる。

## POSTとリクエストボディ

* Pydanticモデルを定義する
* FastAPIが自動でやってくれること

  * JSON → Pythonオブジェクト
  * 型チェック
  * 必須チェック

Pydantic（パイダンティック）とは、**Pythonでデータのバリデーション（検証）と</br>型管理を簡単・安全に行うためのライブラリ**

特にFastAPIと一緒に使われることが多く、API開発では事実上の標準ツール

### 何ができるのか？

Pydanticの主な役割は次の3つ

#### ① データのバリデーション（検証）

入力データが
- 型が正しいか
- 必須項目があるか
- 値の範囲が妥当か
を自動でチェック

#### ② 型安全なデータ管理

Pythonの型ヒント(typing)を使って、
安全で読みやすいコードを書ける

#### ③ JSON ↔ Pythonオブジェクト変換

- dict / JSON → Pythonオブジェクト
- Pythonオブジェクト → dict / JSON

を簡単に変換

## レスポンスモデル

* 「APIはこの形で返します」という宣言
* 余計な項目をレスポンスに出さない
* フロントとの約束事を守れる

## ステータスコード

* 成功でも 200 / 201 を使い分けられる
* FastAPI側で明示的に指定可能

## エラー処理

* HTTPException を raise するだけ
* ステータスコードとメッセージを書く

## async / await

* 非同期処理も書ける
* 外部API呼ぶときとかに効くらしい
* 普通のdefと混ざっててもOK

## テストについて

* FastAPI用のテストクライアントがある
* APIを叩くテストが簡単に書ける

## まとめ

FastAPIは

* **高速**
* **型安全**
* **ドキュメント生成が自動**

という特徴を持ち、
PythonでAPI開発を行う上で非常に実践的なフレームワーク
