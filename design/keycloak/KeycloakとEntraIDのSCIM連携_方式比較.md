# パターンA vs パターンB 比較表

本表は、  
**Keycloak と Entra ID を SCIM で連携する際の実装方式比較**である。

| 観点 | パターンA：OSS SCIMプラグイン | パターンB：SCIMゲートウェイ自作 |
|---|---|---|
| 実装工数 | ◎ 少ない | △ 多い |
| 初期導入スピード | ◎ 早い | △ 遅い |
| 運用コスト | ○ 低い | △ 中〜高 |
| セキュリティ拡張性 | △ 限定的 | ◎ 高い |
| Bearer認証制御 | △ 静的トークン前提 | ◎ 自由に設計可能 |
| 監査・説明責任 | △ 弱い | ◎ 強い |
| ログ・監視 | △ プラグイン依存 | ◎ 自由に設計可能 |
| 障害切り分け | △ 難しい | ◎ 明確 |
| Keycloak依存度 | ◎ 高い | ○ 低い |
| Keycloak更新自由度 | △ 制約あり | ◎ 高い |
| SCIM仕様吸収力 | ○ | ◎ |
| 独自ロジック追加 | × 困難 | ◎ 容易 |
| IdP変更耐性 | △ 低い | ◎ 高い |
| 長期運用適性 | △ | ◎ |
| 小規模PoC | ◎ 最適 | △ オーバースペック |
| 中〜大規模本番 | △ リスクあり | ◎ 推奨 |

---

## 方式選定の考え方（要約）

- **スピード・工数最優先** → パターンA  
- **安全性・将来性・責任分界重視** → パターンB  

---

## 一文での結論例（資料用）

> 本システムは将来的な拡張および監査対応を考慮し、  
> OSSプラグイン依存を避け、SCIMゲートウェイ方式（パターンB）を採用する。
