# APIモックとは

APIモック（API Mock）とは、
**実際のAPIの代わりに振る舞う擬似的なAPI**のことです。

本物のAPIがまだ完成していない、使えない、または使いたくない場合に、  
**決まったレスポンスを返す仮のAPI**として利用されます。

---

## なぜAPIモックを使うのか

### 1. フロントエンドとバックエンドの並行開発
- バックエンド未完成でも、フロントエンド開発が進められる
- API仕様（URL・パラメータ・レスポンス）だけ先に決めて作業可能

### 2. テストの安定化
- 本物のAPIの障害・遅延・データ変動の影響を受けない
- 毎回同じレスポンスでテストできる

### 3. 開発コスト・時間の削減
- 外部API（有料・回数制限あり）を叩かずに開発できる

---

## APIモックの具体例

### 本物のAPI（想定）

```http
GET /api/users/1
```

## APIモックのレスポンス例

~~~json
{
  "id": 1,
  "name": "山田 太郎",
  "email": "yamada@example.com"
}
~~~

→ 実際のDBやサーバー処理はなく、**決め打ちのJSONを返すだけ**

---

## APIモックの作り方（代表例）

### 1. ツールを使う

- Mockoon
- Postman（Mock Server）
- Swagger / OpenAPI Mock
- MSW（Mock Service Worker）（フロントエンド向け）

### 2. コードで作る

- Node.js + Express
- Spring Boot
- FastAPI など

例（Node.js + Express）：

~~~js
app.get('/api/users/1', (req, res) => {
  res.json({
    id: 1,
    name: '山田 太郎',
    email: 'yamada@example.com'
  });
});
~~~

---

## APIモックとスタブ・フェイクの違い（簡単）

| 用語 | 概要 |
|---|---|
| APIモック | API全体を擬似的に再現 |
| スタブ | 決まった値を返す簡易実装 |
| フェイク | 簡易的なロジックを持つ実装 |

※ 実務ではまとめて「モック」と呼ばれることも多いです

---

## 一言でいうと

> **APIモック = 本物のAPIがなくても開発・テストできる“代役API”**
