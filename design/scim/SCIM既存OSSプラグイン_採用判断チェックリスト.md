# パターンA（既存OSS SCIMプラグイン）を選んでよい条件チェックリスト

本チェックリストは、  
**Keycloak + 既存 OSS の SCIM プラグイン（パターンA）を採用しても問題ないか**を判断するためのものである。

すべてを満たす必要はないが、  
❌ が多い場合は **パターンB（SCIMゲートウェイ方式）を検討すべき**である。

---

## 1. システム規模・重要度

- [ ] 利用ユーザー数が少〜中規模（目安：〜数百人）
- [ ] 社内向けシステムで、全社基盤・基幹ではない
- [ ] SCIM同期停止が即ビジネス停止につながらない

---

## 2. セキュリティ・ガバナンス要件

- [ ] OSS利用が社内ルール上問題ない
- [ ] SBOM提出や厳密な監査対応が不要
- [ ] Bearerトークンは静的シークレットで問題ない
- [ ] IP制限、mTLS、署名検証などの追加要件がない

---

## 3. 運用・保守体制

- [ ] OSSプラグインの更新停止リスクを許容できる
- [ ] Keycloak本体アップデートが慎重（頻繁に行わない）
- [ ] 障害時に「OSS由来」と説明できる組織文化がある
- [ ] ログ・監視要件が最低限で足りる

---

## 4. 機能要件

- [ ] Entra ID の標準 SCIM 機能のみを利用する
- [ ] PATCH / filter / Group 同期などに強い要件がない
- [ ] ユーザー属性変換や独自ロジックが不要
- [ ] Realm横断や他システム連携を行わない

---

## 5. 将来計画

- [ ] IdP（Entra ID）変更の予定がない
- [ ] SCIM以外の連携方式を将来追加する予定がない
- [ ] Keycloakへの依存が強くても問題ない

---

## 判定の目安

- ✅ が多い → **パターンAを選択しても問題ない**
- ❌ が複数該当 → **パターンB（SCIMゲートウェイ方式）を推奨**
